# Chapter-detection harness

Iteration loop for `detectChaptersFromText` in `src/lib/pdf.ts`.

## Run

```
bun run test:fixtures   # regenerate synthetic PDFs (run once, or after edits)
bun run test            # run detection against every fixture, print diff
```

The harness scores detected chapters against expected ones with ±2-page
fuzz. Each fixture passes when:
- every expected chapter has a detected match within 1 page,
- at least half of titles share a normalized token, and
- false-positives are ≤ 1.

## Synthetic fixtures

Generated by `test/generate-fixtures.ts` into `test/fixtures/`. Each
exercises a known pattern:

| name         | what it tests                                                        |
| ------------ | -------------------------------------------------------------------- |
| novel        | classic novel, centered roman numerals, same font as body            |
| paper        | arxiv-style sections, bold larger headings                           |
| textbook     | running header on every page, two-line chapter openings              |
| memoir       | small italic section names (Prologue/Chapter/Epilogue)               |
| frontmatter  | front matter offsets body — chapter 1 is at PDF page 5, not 1        |

## Testing against a real PDF

Drop a PDF into `test/fixtures/` alongside a matching `.expected.json`:

```json
// test/fixtures/mybook.expected.json
{
  "description": "One-line description",
  "chapters": [
    { "title": "Chapter 1", "page": 12 },
    { "title": "Chapter 2", "page": 28 }
  ]
}
```

Then `bun run test`. The page numbers are real PDF page numbers, not the
book's printed page numbers.
